# 🎵 Telegram Audio Bot PRO v2.7.6

Профессиональный Telegram-бот для обработки аудио с мягкой компрессией, деэссером и сохранением динамического диапазона.

## ✨ Возможности

- **📊 Детальный анализ** — анализ качества, динамики и частотных характеристик
- **📈 Спектральный анализ** — визуализация формы волны и частотного спектра
- **✨ Мягкое улучшение** — три уровня компрессии (1.5:1, 2.0:1, 3.0:1) с сохранением динамики
- **🔊 Нормализация громкости** — точная нормализация по стандарту LUFS (-16 LUFS)
- **🎙 Деэссер** — подавление сибилянтов (резких "с", "ш") без потери яркости
- **🎵 Конвертация Моно → Стерео** — преобразование моно в стерео
- **💾 Конвертация форматов** — поддержка FLAC, MP3 (320kbps), OGG Vorbis (q10), WAV
- **🚀 Полная обработка** — автоматическая обработка со всеми улучшениями
- **📁 Сервер скачивания** — для файлов >50 МБ автоматически создаётся одноразовая ссылка (TTL 30 мин)

## 🎚️ Что нового в v2.7.6

- ✅ **Сервер скачивания** — файлы >50 МБ отправляются через HTTP-ссылку, не через Telegram
- ✅ **Деэссер** — подавление сибилянтов через sidechaincompress без разрушения тембра
- ✅ **Исправлены имена файлов** — корректное отображение на iOS/Android
- ✅ **Поддержка Cyrillic** — UTF-8 в заголовках Content-Disposition (RFC 5987)
- ✅ **Корректные MIME-типы** — файлы открываются как аудио, не как .bin
- ✅ **Надёжные ссылки** — HTML вместо Markdown, без спецсимволов

## 📋 Требования

- [Docker](https://www.docker.com/get-started) и Docker Compose
- Telegram Bot Token (получите у [@BotFather](https://t.me/BotFather))
- Открытый порт `8765` на сервере (для сервера скачивания файлов)

## 🚀 Быстрый старт

### 1. Клонирование проекта

```bash
git clone https://github.com/Jefferssonn/my-audio-bot.git
cd my-audio-bot
```

### 2. Получение токена бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям и придумайте имя бота
4. Скопируйте полученный токен (например: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 3. Настройка переменных окружения

Скопируйте файл `.env.example` в `.env`:

```bash
cp .env.example .env
```

Откройте `.env` и заполните:

```env
BOT_TOKEN=ваш_токен_здесь
MAX_FILE_SIZE_MB=20
CLEANUP_INTERVAL_MINUTES=30
TEMP_FILE_MAX_AGE_HOURS=2
DOWNLOAD_HOST=ваш_ip_сервера
DOWNLOAD_PORT=8765
```

### 4. Запуск бота

```bash
docker-compose up -d
```

#### Linux/Mac (скрипты):

```bash
./scripts/start.sh    # Запуск
./scripts/stop.sh     # Остановка
./scripts/logs.sh     # Логи
./scripts/restart.sh  # Перезапуск
```

#### Windows (PowerShell):

```powershell
.\scripts\start.bat
.\scripts\stop.bat
.\scripts\logs.bat
.\scripts\restart.bat
```

## 📖 Использование

1. Отправьте команду `/start` боту в Telegram
2. Отправьте аудиофайл (поддерживаются: MP3, WAV, FLAC, OGG, M4A и др.)
3. Выберите нужное действие из меню

### Доступные функции

#### 📊 Анализ
Детальная информация о файле:
- Каналы (моно/стерео), частота дискретизации, битность
- Длительность, размер, качество (%)
- RMS, Peak, динамический диапазон, громкость в LUFS

#### 📈 Спектр
Визуализация: форма волны (waveform) + частотный спектр (FFT)

#### ✨ Улучшение
Три уровня компрессии:
- **Light (1.5:1)** — самая мягкая, для музыки с высокой динамикой
- **Medium (2.0:1)** — рекомендуется для большинства случаев
- **Heavy (3.0:1)** — для подкастов и голоса

#### 🔊 Нормализация
Нормализация громкости до -16 LUFS (стандарт для стриминговых сервисов)

#### 🎙 Деэссер
Подавление резких сибилянтов ("с", "ш", "щ") в голосовых записях и вокале.
Работает через частотно-зависимую компрессию — сохраняет тембр, убирает неприятную резкость.

#### 🎵 Моно → Стерео
Конвертация моно-записи в стерео

#### 💾 Конвертер
- **FLAC** — без потерь (lossless)
- **MP3** — 320 kbps
- **OGG Vorbis** — q10
- **WAV** — PCM без сжатия

#### 🚀 Полная обработка
Автоматически применяет:
1. Конвертацию моно → стерео (если нужно)
2. Мягкую компрессию (Medium 2:1)
3. Нормализацию громкости
4. Экспорт в FLAC

#### 📁 Скачивание больших файлов
Если обработанный файл превышает лимит Telegram (50 МБ), бот автоматически:
1. Размещает файл на встроенном HTTP-сервере
2. Отправляет одноразовую ссылку для скачивания
3. Ссылка действует **30 минут**, затем файл удаляется

## ⚙️ Настройка

### Переменные окружения (.env)

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `BOT_TOKEN` | Токен Telegram бота | обязательно |
| `MAX_FILE_SIZE_MB` | Максимальный размер входящего файла (МБ) | `20` |
| `CLEANUP_INTERVAL_MINUTES` | Интервал очистки temp-файлов (мин) | `30` |
| `TEMP_FILE_MAX_AGE_HOURS` | Максимальный возраст temp-файлов (ч) | `2` |
| `DOWNLOAD_HOST` | IP-адрес сервера для ссылок скачивания | обязательно |
| `DOWNLOAD_PORT` | Порт сервера скачивания | `8765` |

### Ресурсы Docker

В `docker-compose.yml` настроены лимиты:

```yaml
deploy:

      memory: 4G
      cpus: '2'
    reservations:
      memory: 2G
```

## 🐛 Устранение проблем

### Бот не запускается
1. Проверьте, что Docker запущен
2. Проверьте токен в `.env`
3. Посмотрите логи: `docker-compose logs`

### Бот не отвечает
1. Проверьте статус: `docker-compose ps`
2. Проверьте логи: `docker-compose logs -f`
3. Перезапустите: `docker-compose restart`

### Ошибка "Request Entity Too Large (413)"
Файл превышает лимит Telegram (50 МБ). Бот автоматически создаст ссылку для скачивания — проверьте, что `DOWNLOAD_HOST` и `DOWNLOAD_PORT` заданы в `.env` и порт открыт на сервере.

### Ошибки обработки аудио
- Проверьте формат файла (поддерживаются: MP3, WAV, FLAC, OGG, M4A)
- Проверьте размер файла (лимит `MAX_FILE_SIZE_MB` в `.env`)

### Недостаточно памяти
Увеличьте лимит в `docker-compose.yml`:
```yaml
limits:
  memory: 8G
```
